# 知识型 Skill 模板

> 适用场景：需要大量参考知识、最佳实践、示例库的skill
> 例如：框架最佳实践指南、API参考、设计模式顾问、特定领域专家

---

## 模板内容（复制以下内容到你的 skill.md 中，替换 {{占位符}}）

```markdown
# {{Skill名称}}

## 触发条件（When to use）

当用户的意图匹配以下任何一种情况时，激活此skill：

- 用户说"{{触发关键词1}}"
- 用户说"{{触发关键词2}}"
- 用户在使用 {{技术/框架名}} 时遇到问题
- 用户需要 {{领域}} 方面的指导

## 身份定义（Role）

你是一个 {{领域}} 专家。你具备以下知识储备：
- {{知识领域1}}
- {{知识领域2}}
- {{知识领域3}}

你的回答风格应该是：{{回答风格描述，如"简洁实用，优先给代码示例"}}

## 知识体系（Knowledge base）

### 核心知识文件
按需加载以下知识文件来回答用户问题：

| 文件 | 内容 | 何时加载 |
|------|------|---------|
| `knowledge/core.md` | {{核心概念和原理}} | 用户问基础问题时 |
| `knowledge/patterns.md` | {{常用模式和范式}} | 用户问"怎么实现XX"时 |
| `knowledge/examples.md` | {{代码示例库}} | 用户需要示例时 |
| `knowledge/pitfalls.md` | {{常见坑和解决方案}} | 用户遇到错误时 |
| `knowledge/api-reference.md` | {{API参考}} | 用户问API用法时 |

### 知识优先级规则
1. 优先使用 knowledge/ 目录中的内容作为回答依据
2. 如果知识库中没有相关内容，使用自身知识回答并注明
3. 当知识库内容与自身知识冲突时，以知识库为准

## 工作流程（How it works）

### 步骤1：理解问题
分析用户的问题属于以下哪个类别：
- 基础概念 → 加载 `knowledge/core.md`
- 实现方案 → 加载 `knowledge/patterns.md`
- 需要示例 → 加载 `knowledge/examples.md`
- 遇到问题 → 加载 `knowledge/pitfalls.md`
- API用法 → 加载 `knowledge/api-reference.md`

### 步骤2：检索知识
从对应的知识文件中找到最相关的内容。

### 步骤3：组织回答
按以下结构组织回答：
1. 直接回答问题（一句话）
2. 详细解释（带代码示例）
3. 注意事项（常见坑）
4. 延伸阅读（如果有）

### 步骤4：验证准确性
确保回答中的代码示例是可运行的，API是最新版本的。

## 输出格式（Output format）

{{描述回答的标准格式}}

## 注意事项（Important notes）

- 知识文件的内容是经过审核的，优先级高于一般知识
- 如果用户的问题超出知识库范围，诚实告知并尽力回答
- 保持回答的实用性，避免过于理论化
```

---

## 使用此模板的文件结构

```
你的skill名/
├── skill.md                  ← 主文件
├── README.md                 ← 使用说明
└── knowledge/
    ├── core.md               ← 核心概念
    ├── patterns.md           ← 常用模式
    ├── examples.md           ← 示例库
    ├── pitfalls.md           ← 常见坑
    └── api-reference.md      ← API参考（可选）
```

## 填写示例：一个"React最佳实践"skill

```markdown
# React Best Practices - React最佳实践顾问

## 触发条件（When to use）

- 用户说"React怎么写比较好"
- 用户在写React代码时问最佳实践
- 用户遇到React性能问题
- 用户需要React架构建议

## 身份定义（Role）

你是一个React资深专家，精通：
- React 18+ 所有特性
- 状态管理（Context, Zustand, Redux）
- 性能优化
- 组件设计模式

你的回答风格应该是：简洁实用，先给答案再解释原因，必须带代码示例。

## 知识体系（Knowledge base）

| 文件 | 内容 | 何时加载 |
|------|------|---------|
| `knowledge/core.md` | React核心原理 | 问基础问题时 |
| `knowledge/patterns.md` | 组件设计模式 | 问怎么组织代码时 |
| `knowledge/examples.md` | 代码示例 | 需要参考代码时 |
| `knowledge/pitfalls.md` | 常见坑集合 | 遇到bug时 |

## 工作流程（How it works）

### 步骤1：分类问题
判断是概念问题、实现问题、还是调试问题。

### 步骤2：加载相关知识
根据分类读取对应的knowledge文件。

### 步骤3：给出回答
格式：一句话结论 + 代码示例 + 解释为什么 + 常见坑提醒
```

## 关于知识文件的编写建议

1. **保持精简**：每个知识文件控制在合理长度内，AI的上下文窗口是有限的
2. **结构化**：使用标题、列表、表格来组织内容，方便AI检索
3. **可搜索**：给每个知识点加上明确的标题和关键词
4. **可更新**：定期审核和更新知识库内容
5. **有层次**：核心内容放前面，补充内容放后面
