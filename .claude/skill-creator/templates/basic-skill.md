# 基础型 Skill 模板

> 适用场景：纯知识类、纯指令类的skill，不需要执行脚本
> 例如：代码风格检查、文案润色、翻译助手、代码评审

## 填写示例：一个"代码评审"skill

```markdown
# Code Reviewer - 代码评审助手

## 触发条件（When to use）

当用户的意图匹配以下任何一种情况时，激活此skill：

- 用户说"帮我review代码"
- 用户说"检查一下这段代码"
- 用户说"代码评审"
- 用户的需求涉及代码质量检查和改进建议

## 能力说明（What it does）

### 能做的事：
- 检查代码中的bug和逻辑错误
- 评估代码风格和可读性
- 提供具体的改进建议

### 不能做的事：
- 不自动修改用户的代码（只提建议）
- 不评估业务逻辑是否正确（只看代码质量）

## 工作流程（How it works）

### 步骤1：读取代码
读取用户指定的文件或代码片段，理解代码的上下文和目的。

### 步骤2：多维度分析
从以下维度逐一检查：
1. 正确性：是否有bug？
2. 安全性：是否有安全漏洞？
3. 性能：是否有性能问题？
4. 可读性：命名、注释、结构是否清晰？
5. 最佳实践：是否遵循了语言/框架的惯例？

### 步骤3：输出评审报告
按严重程度分类输出发现的问题和改进建议。

## 输出格式（Output format）

按以下格式输出评审结果：

### 严重问题（Must Fix）
- 文件:行号 - 问题描述 - 建议修复方式

### 建议改进（Should Fix）
- 文件:行号 - 问题描述 - 建议修复方式

### 可选优化（Nice to Have）
- 文件:行号 - 问题描述 - 建议修复方式

### 总结
总体评价 + 核心改进方向

## 注意事项（Important notes）

- 只提供建议，不自动修改代码
- 优先指出严重问题，避免吹毛求疵

## 错误处理（Error handling）

- 如果用户没有指定文件，则询问要评审哪个文件
- 如果文件不存在，则提示用户检查路径
```
