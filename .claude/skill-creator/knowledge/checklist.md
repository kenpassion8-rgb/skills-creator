# Skill 质量检查清单

> 创建完Skill后，逐项检查以下内容。全部通过才算合格。

---

## 第一关：文件结构检查

- [ ] skill.md 文件存在且不为空
- [ ] 文件夹名称使用英文小写+连字符（如 my-skill）
- [ ] 没有多余的空文件或临时文件
- [ ] 如果有 tools/ 目录，脚本文件有执行权限（chmod +x）
- [ ] 没有包含敏感信息（密码、密钥、token等）

---

## 第二关：skill.md 内容检查

### 触发条件
- [ ] 有明确的触发条件（不是"随时可用"）
- [ ] 触发条件足够具体（不会误触发）
- [ ] 列出了至少2-3个触发关键词或场景
- [ ] 触发条件覆盖了用户可能的表达方式

### 能力说明
- [ ] 明确列出了"能做的事"
- [ ] 明确列出了"不能做的事"（边界清晰）
- [ ] 能力描述准确，没有夸大

### 工作流程
- [ ] 有清晰的步骤划分（至少2步）
- [ ] 每一步都是可执行的具体动作（不是"分析"、"理解"等模糊词）
- [ ] 步骤之间有逻辑顺序
- [ ] 包含了所有必要的步骤，不会遗漏关键环节

### 输出格式
- [ ] 定义了输出的格式或模板
- [ ] 提供了输出示例
- [ ] 格式清晰易读

### 错误处理
- [ ] 考虑了"用户没给必要输入"的情况
- [ ] 考虑了"文件/资源不存在"的情况
- [ ] 考虑了"权限不够"的情况
- [ ] 每个错误场景都有具体的处理方式

---

## 第三关：知识文件检查（如果有 knowledge/ 目录）

- [ ] 每个知识文件都有明确的主题
- [ ] 内容结构化（使用标题、列表、表格）
- [ ] 没有重复内容（不同文件之间不重叠）
- [ ] skill.md 中说明了什么时候加载哪个文件
- [ ] 知识内容准确可靠

---

## 第四关：工具脚本检查（如果有 tools/ 目录）

- [ ] 脚本有执行权限
- [ ] 脚本有参数验证（缺少参数时报错提示）
- [ ] 脚本有错误处理（set -euo pipefail 或等价机制）
- [ ] 没有硬编码的绝对路径
- [ ] 没有危险的破坏性命令（rm -rf / 等）
- [ ] skill.md 中有脚本的使用说明

---

## 第五关：可用性测试

- [ ] 模拟一个典型场景，skill能正常工作
- [ ] 模拟一个边界场景（如缺少输入），错误处理正常
- [ ] 输出结果符合预定义的格式
- [ ] 整个流程没有断链（每步的输出能作为下一步的输入）

---

## 第六关：文档检查（如果有 README.md）

- [ ] 说明了skill的用途
- [ ] 说明了如何安装
- [ ] 说明了如何使用（包含示例）
- [ ] 说明了前置条件（如果有）
- [ ] 写给人类看的，语言通顺易懂

---

## 快速评分表

| 维度 | 满分 | 自评 | 说明 |
|------|------|------|------|
| 触发精准度 | 20分 | __ | 触发条件是否精确 |
| 流程清晰度 | 25分 | __ | 步骤是否可执行 |
| 输出质量 | 20分 | __ | 输出是否有格式有示例 |
| 容错能力 | 15分 | __ | 错误处理是否完善 |
| 文档完整度 | 10分 | __ | 文档是否齐全 |
| 代码安全性 | 10分 | __ | 脚本是否安全 |
| **总分** | **100分** | __ | 80分以上算合格 |

---

## 常见扣分项

1. **触发条件太宽泛**（-10分）：容易在不该触发时触发
2. **步骤用模糊动词**（-10分）："分析"、"理解"、"处理"没有具体说明
3. **没有输出模板**（-5分）：AI输出格式不统一
4. **没有错误处理**（-10分）：遇到异常就卡住
5. **脚本没有参数验证**（-5分）：缺少参数时行为不可预期
6. **硬编码路径**（-5分）：换个环境就用不了
