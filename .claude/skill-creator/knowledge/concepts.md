# Skill 核心概念 - 从零理解

## 什么是 Skill？

### 一句话解释
Skill（技能包）= 一份写给AI看的"操作手册"，让AI学会一个新本事。

### 生活比喻

想象AI是一个非常聪明但什么都不知道的新员工：
- **没有Skill的AI**：什么都能做一点，但没有专精，每次都要你从头教
- **有Skill的AI**：相当于给这个新员工一本《岗位操作手册》，他按手册做事，又快又准

再比如：
- Skill 就像手机上的 App
- 手机本身（AI）有基础能力
- 装了App（Skill）之后，就有了特定的专业能力
- 你可以装很多不同的App，让手机能做很多事

### 技术定义

Skill 是一组结构化的指令文件，安装在 `~/.claude/skills/` 目录下，告诉 Claude Code：
1. 什么时候激活这个技能（触发条件）
2. 这个技能能做什么（能力范围）
3. 具体怎么一步步做（工作流程）
4. 遇到问题怎么办（错误处理）

## Skill 的组成部分

### 必需部分：skill.md

这是唯一必需的文件，相当于整个技能包的"大脑"。

它包含：
```
skill.md
├── 触发条件    →  "用户说了什么话时，我要出动"
├── 能力说明    →  "我能做什么、不能做什么"
├── 工作流程    →  "一步步怎么做"
├── 输出格式    →  "结果长什么样"
├── 注意事项    →  "有什么要小心的"
└── 错误处理    →  "出问题了怎么办"
```

### 可选部分

| 部分 | 文件/目录 | 作用 | 什么时候需要 |
|------|----------|------|-------------|
| 使用说明 | README.md | 告诉人类怎么用 | 需要分享给别人时 |
| 工具脚本 | tools/ | 可执行的脚本 | 需要执行命令时 |
| 知识库 | knowledge/ | 参考资料 | 有大量专业知识时 |
| 模板 | templates/ | 可复用的模板 | 需要生成格式化内容时 |

## 三种 Skill 类型

### 类型一：基础型（Basic）

**特点**：只需要 skill.md，不需要任何脚本或额外文件

**适合场景**：
- 代码评审（按规则检查代码）
- 文案润色（按风格修改文字）
- 翻译助手（按规则翻译）
- 回答格式化（按固定格式回答）

**文件结构**：
```
my-skill/
└── skill.md      ← 只需要这一个文件
```

**理解方式**：就像给AI一张"操作卡片"，上面写着做事的步骤。

### 类型二：工具型（Tool-based）

**特点**：除了 skill.md，还有可执行的脚本

**适合场景**：
- 自动部署（需要执行deploy脚本）
- 数据处理（需要运行数据脚本）
- 项目初始化（需要执行创建命令）
- 日志分析（需要运行分析脚本）

**文件结构**：
```
my-skill/
├── skill.md
└── tools/
    └── do-something.sh
```

**理解方式**：就像给AI一个"工具箱"，里面有各种工具，skill.md 告诉AI什么时候用哪个工具。

### 类型三：知识型（Knowledge-heavy）

**特点**：除了 skill.md，还有大量参考知识文件

**适合场景**：
- 框架最佳实践（React/Vue/Angular专家）
- API参考指南（某个服务的API用法）
- 设计模式顾问（各种设计模式的知识）
- 特定领域专家（医学/法律/金融知识）

**文件结构**：
```
my-skill/
├── skill.md
└── knowledge/
    ├── core.md
    ├── patterns.md
    └── examples.md
```

**理解方式**：就像给AI一个"参考书架"，skill.md 告诉AI遇到什么问题去翻哪本书。

## Skill 的生命周期

```
创建 → 安装 → 使用 → 维护 → 升级/退役

创建：按模板编写文件
安装：放到 ~/.claude/skills/ 目录
使用：AI自动识别触发条件并执行
维护：定期检查效果，修复问题
升级：根据反馈优化内容
退役：不再需要时删除文件夹
```

## 常见误解

**误解1**："Skill需要写很多代码"
**事实**：最简单的Skill只需要一个skill.md文件，用自然语言写就行，不需要任何代码。

**误解2**："Skill一旦创建就不能改了"
**事实**：Skill随时可以编辑修改，改了立即生效。

**误解3**："一个Skill应该尽量多做事"
**事实**：好的Skill只做一件事，做到极致。如果需要多个能力，创建多个Skill。

**误解4**："Skill需要联网才能用"
**事实**：Skill是本地文件，不需要联网。除非你的Skill明确需要调用网络API。
